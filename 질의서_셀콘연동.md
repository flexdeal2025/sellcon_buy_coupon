# 📨 셀콘 세션에 보내는 연동 질의서 (재고 직결 설계용)

> 목적: 컨트롤타워 `POST /api/stock/ingest` 계약을 확정하기 위해, 셀콘이 매입 시점에
> 무엇을 보유/제공할 수 있는지 확인. 🔴=계약 확정에 필수 / 🟡=있으면 좋음
>
> ⏩ **가장 빠른 답변법**: 아래 개별 답 대신, (1) 셀콘의 **매입·정산 관련 테이블 스키마**(컬럼명·타입)와
> (2) **매입 1건의 샘플 레코드(JSON, 개인정보 마스킹)** 를 주시면 대부분 자동 해소됩니다.

---

## A. 매입/쿠폰 데이터 모델

- 🔴 **A1.** 매입 확정 시점에 셀콘 DB에 저장되는 값은? 특히 `coupon_code`, `expiry_date`(형식?), `상품명`, `옵션명`, 교환처(브랜드).
- 🔴 **A2.** 코드형/이미지형을 셀콘은 어떤 필드로 구분하나? (한 매입에 코드·이미지가 섞일 수 있나?)
- 🟡 **A3.** `coupon_code` 형식·길이 규칙(브랜드별 상이?)·공백/하이픈 포함 여부.
- 🟡 **A4.** `expiry_date`를 셀콘이 항상 확보하나, 아니면 일부 누락 가능한가? (누락 시 우리 검수로 넘김)

## B. 정산·지급(증빙) 데이터 — 세무 소명 핵심

- 🔴 **B1.** **`payout_uuid`(정산 건 고유 ID)가 존재하나?** 없으면 매입건당 안정적 고유 ID는 무엇?
- 🔴 **B2.** **지급(payout)과 쿠폰의 관계: 1:1인가, 1:다(한 번 지급에 여러 쿠폰)인가?**
  → 멱등 키 단위를 가른다(아래 C1).
- 🔴 **B3.** **`payout_amount`(매도자 실지급액)와 우리가 쓸 매입원가(`unit_cost`)가 같은가?**
  셀콘이 수수료·보너스(첫 판매 1,000원)를 떼거나 더하면 둘이 달라짐 → 분리 전송 필요.
- 🔴 **B4.** **매도자 식별정보**를 어디까지 제공 가능한가? `seller_name`(실명), 연락처, 계좌·은행(마스킹).
  (개인정보 공유 정책·마스킹 수준 포함) — 원천징수 누적추적·소명에 필요.
- 🟡 **B5.** `payout_date`(송금일), `payout_method`(계좌이체/토스/카카오페이 등), 이체 참조번호 제공 가능?
- 🟡 **B6.** 매도자 동의(개인정보·거래) 기록이 셀콘에 남나? 소명 시 캡처/근거로 끌어올 수 있나?

## C. 멱등 키 · 트리거 타이밍 · 취소

- 🔴 **C1.** **쿠폰 1건당 안정적 고유 ID**(예: `sellcon_item_id`)가 있나? — 멱등 `source_ref`를 이걸로 만든다.
  (B2가 1:다면 payout 단위가 아니라 **쿠폰 단위 ID**가 반드시 필요)
- 🔴 **C2.** 푸시 **트리거 시점**: 매도자 제출 / 매입 확정 / 지급(정산) 완료 중 언제? 상태 라이프사이클이 있나?
- 🔴 **C3.** **취소·반품**: 우리에게 푸시한 뒤 셀콘 쪽에서 그 매입이 취소/철회되면 어떻게 알리나?
  (우리 발행 전이면 스테이징에서 빼야 함 → 철회 통지 수단 필요. 발행 후는 별도 협의.)
- 🟡 **C4.** 같은 쿠폰이 재전송될 수 있는 상황(네트워크 재시도 외)이 있나?

## D. 이미지 접근 (이미지형)

- 🔴 **D1.** 이미지 저장소는? (Supabase Storage / GCS / S3 …) **서명 URL 발급이 가능**하고 **TTL ≥ 1시간** 설정 가능한가?
- 🟡 **D2.** 이미지 형식(jpg/png)·평균 용량·쿠폰당 장수(1장?).
- 🟡 **D3.** 서명 URL 대신 바이트 직접 전송을 선호하나? (우리는 URL 참조 권장 — 가벼움)

## E. 인증 · 인프라 · 볼륨

- 🔴 **E1.** 셀콘은 **서버사이드에서 외부로 POST**가 가능한가(브라우저가 아니라 서버 훅)? 호스팅 환경(Vercel/기타)·아웃바운드 제약?
- 🔴 **E2.** `STOCK_INGEST_KEY`(공유 비밀)를 셀콘 `.env`에 보관하고 헤더로 보낼 수 있나? (키 발급·공유 방법 합의)
- 🟡 **E3.** 예상 **볼륨**: 하루 매입 건수·피크, 1회 정산당 평균 쿠폰 수. (배치·레이트 설계)

## F. 상품 매핑

- 🔴 **F1.** 셀콘의 상품명이 우리 스마트스토어 상품(`channel_product_no`/상품명·옵션)과 **매핑되나**, 아니면 자유 텍스트인가?
  매핑 키가 있으면 자동연결, 없으면 우리 검수에서 수동매칭(텔레그램 봇의 `product_option_map` 재사용 가능).
- 🟡 **F2.** 셀콘 상품 ID(`sellcon_item_id` 등)와 우리 상품의 대응표가 있나/만들 수 있나?

---

## ✅ 우리가 가장 알고 싶은 3가지 (이것만이라도 먼저)
1. **B2 + C1** — 지급↔쿠폰이 1:다인지, **쿠폰 1건당 고유 ID**가 있는지 (멱등 설계의 핵심).
2. **B3 + B4** — `payout_amount` vs 원가 차이 여부 + **매도자 실명/연락처 제공 가능 범위** (세무 소명).
3. **C2 + C3** — **언제 푸시**하고, **취소되면 어떻게 알리는지** (오적재 방지).
