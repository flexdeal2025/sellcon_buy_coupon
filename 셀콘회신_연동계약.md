## ① 셀콘이 할 일 (작업 목록)

1. **동의완료 훅 → ingest 호출**: `gifticons.salesConsentAt`가 찍히는 순간(판매 동의 완료 = 소유권 이관) `POST {타워}/api/stock/ingest` 호출. (검수가 항상 선행하므로 이때 코드·유효기간·이미지 확정 상태)
2. **'재고 전환 완료' 자동 태깅**: 위 호출의 **성공 응답(`ok:true` + `id`)** 을 받으면 셀콘 '매입 현황'에 **'재고 전환 완료' 태그 + `towerStockId(=응답 id)`** 저장. `deduped:true`도 동일 처리. → 기존 관리자 수동 태깅을 대체. **수동 전환(예외건)도 같은 API를 거쳐** 태깅 일원화.
3. **취소 철회 호출**: 동의완료 후 매입이 `CANCELLED`/`REJECTED`로 바뀌면 `DELETE {타워}/api/stock/ingest?source_ref=…` 호출(미발행분 철회).
4. **(Phase 2) 정산 스냅샷**: `PAYOUT_COMPLETED` 시 `POST {타워}/api/stock/ingest/payout` 호출 — 확정 지급일 + `kyc_name_hash`(셀콘이 SHA-256 해시 계산해 전송, 평문 금지) 전달. 내구성 봉인용.
5. **인증/인프라**: `STOCK_INGEST_KEY`를 셀콘 `.env`에 보관, 헤더 `Authorization: Bearer <KEY>`. 응답을 기다려 태깅(타워 응답은 빠름). 네트워크 실패 시 재시도(멱등 안전).

## ② 최종 JSON 계약

### `POST /api/stock/ingest`  (헤더: `Authorization: Bearer <STOCK_INGEST_KEY>`)

**요청** (필드 오른쪽 = 셀콘 출처)
```jsonc
{
  "source": "sellcon",
  "source_ref": "sellcon_gifticon_<gifticons.id>",   // 멱등키(쿠폰 1건당 고유). 재시도해도 1건
  "stored_as_code": true,                  // couponCode 있으면 true, 없으면 false(이미지형)
  "product_name": "스타벅스 아메리카노 Tall", // products.title
  "option_name": "",                       // 비우면 타워 기본값('유효기간 최소 10일 이상…')
  "coupon_code": "1234-5678-9012",         // gifticons.couponCode (이미지형이면 생략/null)
  "expiry_date": "2026-09-30",             // gifticons.validPeriod → KST 날짜(YYYY-MM-DD)
  "image_url": "https://….supabase.co/…/uuid.jpg", // gifticons.imageUrl (퍼블릭 URL OK)
  "supplier": "셀콘",

  // ── 핵심 증빙 ──
  "purchase_channel": "sellcon_auto",
  "proof_type": "no_formal_sellcon",
  "payout_uuid": "<gifticons.id>",
  "unit_cost": 38000,                      // purchasePrice = 매입원가
  "payout_amount": 39000,                  // purchasePrice + (bonusAmount ?? 0) = 실지급
  "bonus_amount": 1000,                    // gifticons.bonusAmount (선택)
  "purchase_date": "2026-06-24",           // 검수완료(매입확정)일

  // ── 매도자: 평문 PII 금지 ──
  "seller_ref": "<gifticons.sellerId>",    // 불투명 링크키(PII 아님). 원천징수 누적·소명 조회용
  "seller_name_masked": "홍*동"             // 마스킹명만. 실명·계좌·전화 평문 전송 금지
}
```

**응답**
```jsonc
{ "ok": true, "id": "<towerStockId>", "batch_no": "SC-260624-001", "deduped": false } // 신규
{ "ok": true, "id": "<기존 id>",      "batch_no": "SC-260624-001", "deduped": true  } // 재시도
{ "ok": false, "error": "unauthorized" }  // 401 (키 불일치)
```
→ 셀콘은 `ok:true`(deduped 무관) 받으면 '재고 전환 완료' 태깅 + `id` 저장.

### `DELETE /api/stock/ingest?source_ref=sellcon_gifticon_<id>`  (철회)
```jsonc
{ "ok": true, "withdrawn": true }                       // 미발행분 삭제 완료
{ "ok": true, "withdrawn": false, "reason": "published" } // 이미 발행됨 → 별도 협의(실재고)
```

### `POST /api/stock/ingest/payout`  (Phase 2, 정산 스냅샷·봉인)
```jsonc
{ "source_ref": "sellcon_gifticon_<id>", "payout_date": "2026-06-24",
  "kyc_name_hash": "<SHA-256(kycName)>" }   // 셀콘이 해시 계산. 평문 금지
```

## ③ 핵심 규칙 (왜 이렇게 하나)
- **멱등**: `source_ref`로 1건 보장(지급↔쿠폰 1:1·`gifticons.id` 안정). 재시도 안전.
- **금액 분리**: `unit_cost`(원가) vs `payout_amount`(실지급=원가+보너스) 별도 → 손익 왜곡·이중계상 방지.
- **PII 최소**: 타워는 `seller_ref`+마스킹명만. 실명·계좌는 셀콘 보관, 소명·원천징수 신고 때만 `payout_uuid`로 조회. (동일 법인 = 내부 이전)
- **자동은 검수대기까지만**: 타워는 `pending`으로만 적재, 실판매 발행은 타워 관리자가 수동(오배송 방어).
- **이미지**: 1차는 퍼블릭 URL fetch. 보안 강화 시 셀콘이 `createSignedUrl(TTL≥1h)` 추가(후속).

---

## ④ 협의 필요(키 교환 시 채울 값)
- `STOCK_INGEST_KEY` 발급·공유 방법.
- 타워 base URL(`{타워}` 자리).
- 발행 후 취소(실재고 회수) 정책 — DELETE가 `published` 거부 시 처리 방법.
- (확인) 동의완료 시 코드·유효기간 확정 보장 → **확인됨**(검수가 동의보다 항상 선행).

> 셀콘 측 ①~③ 구현 + 타워 측 엔드포인트가 준비되면 실거래 1건으로 연결 테스트.
